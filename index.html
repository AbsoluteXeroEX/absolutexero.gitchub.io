"What is Starbound and why should I care?"

Well, unless you like indie (independent) games, you probably shouldn't care. However, if you're interested in why I listed this as a project, please continue reading.


Starbound is an extraterrestrial sandbox adventure game.

"How is this relevant to your skills?"

I'm glad you asked! Starbound's developers have made it possible for its players to build and take part in custom content for the game. Custom assets are created using JSON and Lua scripting, both of which are versatile and easy to use. JSON (JavaScript Object Notation) is a lightweight data-interchange format that is easy for people to read and write, as well as easy for machines to parse and generate. As its name implies, JSON is based on a subset of the JavaScript Programming Language. JSON itself is a text format that is completely language-independent, but it also uses conventions that are familiar to programmers of the C-family of languages, including C, C++, C#, Java, JavaScript, Perl, Python, and many others.[What is JSON?]

"Okay, so JSON sounds very helpful, but what about Lua?"

Ah, yes, I almost forgot! Lua is a powerful, efficient, lightweight, embeddable scripting language that is used in many industrial applications, such as Adobe Photoshop Lightroom. It has an emphasis on embedded systems and popular games such as World of Warcraft, Final Fantasy XIV, and Angry Birds. For game developers, Lua is almost a requirement due to its fluidity. In the realm of interpreted scripting languages, Lua is not only the fastest in fine-tuned benchmark programs, but also in real-life applications. Its portability adds to its swiftness in processing, and makes it the ideal scripting language for just about anything!

"So it's safe to assume that this focuses on your content development skills?"

Absolutely! While this mod is technically a continuation of someone else's project (the previous author of the content had to drop it for real-life obligations), I was able to learn a lot about how JSON is structured, and continuously learning about Lua and how it can be used for practical purposes.

"What is Second Life and why should I care?"

Second Life is an online, open sandbox game. Users can create objects, script, and texture as they see fit. Second Life has been in play as far back as 2003; it is operated by Linden Lab and continues to run to this day. Second Life's functionality has improved substantially over the years, so much to the point that it has its own market and currency; players can buy, sell, and trade their creations for the virtual currency, "Linden Dollars" (stylized as "L$"), which can be converted back to U.S. Dollars. Content creation is the forefront of Second Life and has enabled many users to make a very lucrative living off of their marketplace products alone.

"How is this relevant to your skills?"

To be frankly honest, if it weren't for Second Life, I never would have picked up an interest in coding. Second Life's functionality goes beyond just creating objects; it can be done to such a degree that anything can be made interactive. Second Life uses an internal scripting engine with its own language (Linden Scripting Language, or "LSL" for short). LSL's capabilities are a bit limited (at least when compared to other scripting languages), but simple-enough creations can become spectacular because of it!


To give an idea of how interesting it is, Second Life has its own built-in combat system. It's great for virtual military communities, but for RPG afficionados, it's not so great. With the help of LSL, users can create their own in-world combat system separate from the built-in one (the built-in combat system can be switched off by region owners in their own regions). I am in the process of creating my own operating system for one such RPG community, dubbed 'TCS' for short. But because of LSL's limitations, I unfortunately ran out of script memory to add in everything we wanted to. To offset this, I began working on a way to store and call data from an external SQL database. Thanks to certain LSL functions, I'm able to retain and store data for the combat system with little latency in the event one of our scripts crashes!

"Wait, you did what!?"

There are certain functions within LSL that allow it to communicate with web servers and grab data from web pages. Using this same principle, I applied the functions to use PHP querries for a MySQL database server on the community's website! As I said earlier, if it weren't for this, I probably wouldn't be as into coding as I am now. LSL's structure is very similar to JavaScript, which gives me a leg up in some instances.

"Wait, you have another website besides this one?"

I sure do! If you want more details on it, you can click on the "Thea RP Community" project panel.
